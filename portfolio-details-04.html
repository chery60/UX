Redesigning Food.com Website-UX Case Study

Client : Portfolio Project
Category : UX Case Study
Date : 14/April/2020
Website: : www.food.com

Food.com has a massive collection of recipes that are submitted, rated, and reviewed by people who are passionate about food. From international cuisines to quick and easy meal ideas, Food.com is where you can find what you are craving.On the website, users can add reviews, modifications, questions, and photos to over 500,000 recipe pages.

We have observed that users have encountered many issues with the current Food.com website. Getting a cooking video based on the available ingredients and the absence of ordering the required product was given a difficult experience for the user. Due to its complex process, end users found it difficult to understand. Therefore, users needed to scroll through the entire website to know how which recipe can be made at present with the ingredients which are present at home.

So we decided to give the best possible solution where users can get cooking videos based on the availability of the ingredients and add a shopping cart to order the required ingredients without any hassle.

In this process, I have done the -
1) Problem analysis-empathizing with the user by doing so user interviews and understanding the Pain points.
2) Focused on-
What is the problem?
Why is the problem happening?
When is the problem happening?
Where is the problem happening?
How we can solve this problem?
3) Did the heuristic evaluation based on the Nielsen Norman principles.
4) Did the Competitive Analysis.
5) Did the Feature Matrix based on the priorities.
6) Build the Personas.
7)Did the Empathy Mapping.

8) Built the Information Architecture.
9) Built the User Flows.
10) Build the Content Engagement Matrix.

11) Designed the Low fidelity Wireframes.
12) Did the user testing.
13) Build the high fidelity Wireframes.
